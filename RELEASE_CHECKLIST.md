# 数独求败发布核对清单

每次发布新版本必须同时核对以下四层，不能只确认文件已经上传：

1. 代码层：`index.html`、`app.js`、`styles.css` 和版本号完全一致。
2. 档案层：旧版浏览器档案完成迁移，新功能有明确默认值。
3. 开关层：已解锁功能与用户开关分别显示，首次使用采用正确默认状态。
4. 线上层：GitHub Pages 已加载新版本文件，并在实际网页逐项操作验证。

## v2.0 回归项目

- 五级题目、计时、三次错误限制、提示和完成检测。
- 功能权限按当前所玩题目等级计算，不按玩家最高等级计算。
- 初级：快速填充整体关闭。
- 中级：行、列或九宫只剩一个空格时，点击自动填入。
- 高级及以上的基础快速填充：继承中级的行、列、九宫末格填充。
- 高级及以上的扩展快速填充：数字末格提醒及最后九格逐个填充。
- 高级及以上：某数字只剩最后一格时出现提醒，点击填入。
- 高级及以上：最后九格逐个填充，不突然结束。
- 候选标记、同行同列同宫自动清理、错误输入保留原标记。
- 玩家档案、昵称、头像、纪录、技巧次数和勋章。
- 五级积分、双倍扣分、技巧奖励、纪录翻倍和奖励音效。
- 积分升级、越一级挑战、按当天统计的失败降级。
- 手机和电脑尺寸下，棋盘、数字键及操作按钮无需异常滚动。
- “本局挑战”通关率提示在整局中持续显示，不自动消失。
- 题目等级按所需技巧路线划分，不再仅按挖空数量划分；高级题须来自高级技巧题库。
- 一键标记仅高级以上开放：高级扣1000、王者扣2000、求败扣5000，并标记所有空格的合法候选数。
- 本局重点技巧可点击选择；再次点击提示后讲解用法与当前条件，高级以上提示按一键标记标准扣分。
- 普通标记和擦除从初级免费开放；手动标记为连续模式，选择一次数字后持续标记，直到切换数字或关闭标记。
- 标记支持双向连续操作：先选空格可连续添加多个候选，先选数字可连续标记多个空格。
- 先选空格再开启标记时必须保留该空格，并直接进入同格多数字连续标记模式。
- 新一盘默认延续上一盘实际选择的题目等级；只有发生降级时才切换到降级后的等级。
- 双击空格等同开启或关闭标记，且不得因双击第一下误触快速填充。
- 新局首次填数显示“本步解析”时不得造成页面跳动，解析区域需预留空间并关闭滚动锚定。
- 所有等级都必须存在人脑可执行的逻辑突破口，求败局也不得依赖无限猜测或暴力搜索。
- 如使用假设反证，最多允许三层候选传播便出现明确矛盾；超过三层的题不得进入任何正式题库。
- 难度来自技巧种类、技巧组合和识别难度，不来自无止境推演。
- 正式题库模板必须通过 `puzzle-audit.js` 全盘逻辑求解验证，并记录实际使用技巧；未解完或技巧不匹配的模板不得发布。
- 快速连续点击不同格子进行标记时，各格操作不得互相取消；双击识别计时必须按单元格独立。
- 数字连续标记模式必须为幂等添加：同一格重复触发不得把刚添加的候选反向删除。
- 数字连续标记必须即时写入，不得等待双击识别延迟或使用后续已变化的选中数字。
- 区块排除、数对、X-Wing等候选删除技巧不得误标为某个数字的直接填入理由；王者局必须体现多技巧组合实战。
- 王者题至少包含三种进阶技巧、十次进阶关键排除，并分布于前中后阶段；不得一次特殊排除后通篇只剩唯一数。
- 所有题库分级、技巧权重和验收标准以 `PUZZLE_DESIGN.md` 为锁定基准；未生成完整技巧路径证明的题不得发布。
- 题目必须同时满足本级挖空范围和技巧路径要求，任何单项达标都不能入库。
- 所有标记模式均禁止在同行、同列或同宫已存在该数字时重复标记；此规则适用于所有开放标记的等级。
- 候选排除类技巧在有效教学提示出现时立即计入本局技巧、使用次数和完成页，不依赖后续填数识别。
- X-Wing必须存在实际可删除候选才判定为可用，并明确显示数字、两行、两列及删除位置。
- 中级题要求区块排除或数对参与；高级题避免依赖王者级链式推理，维持数对、三数组和适量X-Wing难度。
- 显性数对按初级基础、隐性数对按中级基础处理；王者题不得仅靠单值和数对解完。
- 王者题开局不得直接出现显性唯一或隐性唯一，必须先做候选排除才能释放可填写数字。
- 求败题必须在基础技巧及X-Wing之后仍然形成阻断，并由专业评级证明需要翼、链或ALS等组合技巧。
- 三数组等候选删除技巧不得只在最终填数时检测；删除有效候选或填写由该排除释放的数字时都必须触发技巧记录、解锁提示与奖励。
- “思”互动必须只作为解释和追问入口，不得直接替玩家填数；本地教练回答要基于当前提示和棋盘候选，且明确不联网。
- 双语切换只改变显示语言，不得改变题库、积分、等级、快速填充、标记、提示次数和玩家档案数据。
